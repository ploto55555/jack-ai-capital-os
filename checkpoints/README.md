# Project Checkpoints

This directory preserves structured checkpoints created before major stage changes or chat rotation.

## File Naming

`YYYY-MM-DD_checkpoint-NNN_topic.md`

## Required Sections

1. objective
2. confirmed decisions
3. exact rules and parameters changed
4. datasets and source files used
5. backtests performed
6. results and limitations
7. rejected ideas and reasons
8. unresolved assumptions and conflicts
9. code and files changed
10. relevant Git commits
11. next action

## Chat Rotation Rule

Before recommending a new chat, the assistant must create or update a checkpoint, Current Status, Decision Log, and Latest Handover, then verify the GitHub writes succeeded.
